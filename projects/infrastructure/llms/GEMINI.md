# Gemini Repo Context - infrastructure

Objetivo: dar ao Gemini um bootstrap fino para operar na slice de
`infrastructure` sem duplicar a fonte canônica do workspace.

Consulta obrigatória:

1. `GEMINI.md`
2. `llms/GEMINI.md`
3. `projects/infrastructure/PROJECT_CONTEXT.md`
4. `projects/infrastructure/GIT_HISTORY.md`

Ao atuar neste repo:

- não existe repo real separado; operar nos diretórios canônicos do
  `jpglabs/docs` ligados a infraestrutura
- usar `projects/infrastructure/PROJECT_CONTEXT.md` como resumo estável
- usar `projects/infrastructure/GIT_HISTORY.md` para entender lanes e sinal recente
- registrar histórico específico em `projects/infrastructure/sessions/` quando houver material novo
