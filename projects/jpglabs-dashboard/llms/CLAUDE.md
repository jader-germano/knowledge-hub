- seguir o bootstrap compartilhado do workspace antes de editar
- tratar o dashboard como superfície operacional local-first
- manter a fonte de dados em `src/data/roadmap.ts` consistente com o que o dashboard mostra
- usar a lane Figma apenas quando houver arquivo ou node concreto; sem isso, registrar handoff textual
- não misturar esta superfície com o bootstrap de outros repos
- operar agentes como squad de apoio técnico, com autoria humana preservada no
  commit
- registrar participação de IA em handoff, ADR ou PR description quando
  necessário, sem usar `Co-Authored-By` por padrão
